# 🧠 AI Memory System

> 一个面向个人的、轻量级的、可定制的 AI 个人知识与记忆系统。基于 Obsidian + Claude Code + CC Switch，搜索效率提升 161 倍。

## 🎯 解决什么问题？

每次开新对话，AI 就失忆了。

我之前跟它聊的、改的、踩的坑，全忘了。我得重新解释一遍上下文，它才能接上。

就像跟一个每天早上都失忆的朋友聊天——你得每天早上重新自我介绍一遍。烦。

于是我花了 3 小时，搭了一套 AI 记忆系统。

---

## 🏗️ 三层架构

```
V1 知识库          V2 跨会话记忆         V3 快速搜索
飞书文档 → 结构化    vault → 30秒恢复      索引 → 0.06秒搜索
   ↓                   ↓                    ↓
 信息沉淀            上下文恢复             检索效率
```

## 📖 V1：知识库搭建

> 痛点：散落的飞书文档无法高效检索，学完就忘。

**方案**：通过爬虫批量获取 130+ 篇飞书文档，基于 Wiki 方法论（Schema 定义 → 骨架搭建 → 批量 Ingest → Lint 检查）整理为 145 个结构化页面，覆盖 JS/Vue/React/CSS/网络/安全/性能优化/小程序/工程化 9 大技术领域。

**效果**：5 小时完成全部整理，形成可搜索、可浏览的前端面试知识库。

## 🔗 V2：跨会话记忆

> 痛点：每次 AI 会话「失忆」，需重复交代背景。

**方案**：借鉴 Codex 的 Vault 结构，设计跨会话的持久记忆系统。

- `MEMORY.md` — 总索引，每次会话启动先读它
- `goals.md` — 当前优先级和进行中的任务
- `decisions/` — 重要决策记录（选了什么、为什么、放弃了什么）
- `reference/` — 参考资料
- `projects/` — 项目笔记

**效果**：30 秒接上上下文，AI 从「失忆」到「记住你是谁」。

## 🔍 V3：快速搜索

> 痛点：V2 缺少「快速回顾」能力，无法快速找到之前会话的具体内容。

**方案**：复用 CC Switch 会话管理能力，将存储成本转嫁至现有工具，自身仅构建索引层。

- **动态索引**：对话过程中实时生成
- **离线索引**：定期执行 Shell 脚本，扫描 CC Switch 会话日志（JSONL）
- **增量更新**：通过 diff 比较，只处理新增/修改的文件

**量化验证**：

| 指标 | 原始方法 | 索引系统 | 提升 |
|------|----------|----------|------|
| 搜索时间 | ~10 秒 | 0.062 秒 | **161 倍** |
| 步骤数 | 4 步 | 1 步 | **75% 减少** |
| 搜索范围 | 所有 JSONL | 只搜索索引 | **90%+ 减少** |

## ✨ 特性

- 🚀 **快速搜索**：0.062 秒完成搜索，效率提升 161 倍
- 🔒 **零依赖**：纯 Markdown 文件，不依赖任何插件、服务、数据库
- 🎛️ **完全可控**：我写什么，它就记什么；我改什么，它就存什么
- 🔄 **增量更新**：通过 diff 比较，只处理新增/修改的文件
- 💰 **成本转嫁**：复用 CC Switch 会话管理，存储成本由厂商承担
- 📦 **可迁移**：换个 AI 工具？把文件夹复制过去就行
- 🎨 **可视化**：用 Obsidian 打开，图谱视图一目了然
