# 记忆 Vault 地图

> 本文件由系统自动加载。每次会话开始，读此文件接上上下文。

## 会话启动流程

1. 读 MEMORY.md（本文件）—— 获取 vault 全貌
2. 读 goals.md —— 了解当前优先级
3. 按需查阅其他文件

## 文件索引

| 文件 | 用途 | 更新频率 |
|------|------|----------|
| goals.md | 当前焦点、待办、进行中的任务 | 每次会话 |
| lessons_learned.md | 踩过的坑和解决方案 | 踩坑后 |
| user_preferences.md | 沟通风格、工作习惯 | 发现新偏好时 |
| projects/ | 项目笔记（每个项目一个文件） | 项目进展时 |
| decisions/ | 重要决策记录 | 做决策时 |
| reference/ | 参考资料 | 按需 |

## 使用规则

1. **会话开始**：读本文件 + goals.md，接上上下文
2. **做完一件事**：更新 goals.md，相关结论写进对应文件
3. **踩到坑**：立即写 lessons_learned.md
4. **做技术决策**：写 decisions/ 目录，记下「选了什么、为什么、放弃了什么」
5. **创建新文件**：更新本文件的索引表
