# 闭包

## 核心概念

闭包是指一个函数能够访问其外部函数作用域中的变量，即使外部函数已经执行完毕。

## 原理/机制

1. **词法作用域**：函数在定义时确定作用域，不是调用时
2. **作用域链**：内部函数可以访问外部函数的变量
3. **垃圾回收**：被闭包引用的变量不会被回收

## 代码示例

```javascript
function createCounter() {
    let count = 0;  // 外部函数的变量
    return {
        increment: () => ++count,
        getCount: () => count
    };
}

const counter = createCounter();
counter.increment();  // 1
counter.increment();  // 2
counter.getCount();   // 2
// count 变量在 createCounter 执行完毕后仍然存在
```

## 面试考点

### Q: 闭包的优缺点？
**优点**：
- 数据私有化（封装）
- 状态保持（记忆化）

**缺点**：
- 内存泄漏风险（变量无法回收）
- 性能开销（作用域链查找）

### Q: 如何释放闭包占用的内存？
将引用闭包的变量设为 `null`，让垃圾回收器回收。
